# `@aptos-labs/confidential-asset-bindings` — Usage Examples

All examples use the public API:

```typescript
import {
  solveDiscreteLog,
  batchRangeProof,
  batchVerifyProof,
} from '@aptos-labs/confidential-asset-bindings';
```

No platform-specific imports are required. The library selects the correct implementation (WASM or native) automatically.

---

## 1. Decrypting a confidential balance

A confidential asset balance is stored on-chain as a compressed Ristretto255 point of the form `n * G`. To recover the underlying integer `n`, pass the 32-byte point to `solveDiscreteLog`.

```typescript
import { solveDiscreteLog } from '@aptos-labs/confidential-asset-bindings';

/**
 * Decrypts a confidential balance from an on-chain Ristretto255 point.
 *
 * @param encryptedPoint - 32-byte compressed Ristretto255 point from the chain.
 * @param maxBits - Search space bit width. Use 16 for balances up to 65535,
 *                  or 32 for balances up to ~4.29 billion. Must be 16 or 32.
 */
async function decryptBalance(
  encryptedPoint: Uint8Array,
  maxBits: 16 | 32 = 32,
): Promise<bigint> {
  const balance = await solveDiscreteLog(encryptedPoint, maxBits);
  console.log(`Decrypted balance: ${balance}`);
  return balance;
}

// --- Usage ---
// Suppose your Aptos SDK call returned this 32-byte encrypted balance:
const encryptedFromChain = new Uint8Array([
  0x3e, 0x4f, 0xab, 0x12, 0x67, 0xd1, 0x09, 0xc3,
  0x57, 0x8a, 0x2e, 0x6f, 0x14, 0xb8, 0x44, 0x9d,
  0x72, 0x11, 0xcc, 0x0a, 0x5b, 0x37, 0xe8, 0x92,
  0xf0, 0x03, 0x7d, 0x4a, 0x25, 0xc9, 0x88, 0x1e,
]);

const balance = await decryptBalance(encryptedFromChain, 32);
```

If no solution exists within `[0, 2^maxBits)` the promise rejects. This means either the wrong search space was specified or the point was not generated by the expected scheme.

## 2. Proving a transfer amount is in range

Before submitting a confidential transfer on-chain you must prove the transfer amount is non-negative and within the valid range. This example generates a proof for a single value and then verifies it locally before submission.

```typescript
import { batchRangeProof, batchVerifyProof } from '@aptos-labs/confidential-asset-bindings';

// These base points must match the ones registered on-chain for your asset.
// Shown here as placeholder values — replace with the real points.
const VAL_BASE = new Uint8Array([
  0x86, 0x62, 0x34, 0x74, 0xf5, 0x24, 0x56, 0x11,
  0x66, 0xb8, 0xad, 0x7a, 0x9b, 0x52, 0x7a, 0x5b,
  0x4c, 0x3d, 0x2e, 0x1f, 0x80, 0x71, 0x62, 0x53,
  0x44, 0x35, 0x26, 0x17, 0x08, 0xf9, 0xea, 0xdb,
]);

const RAND_BASE = new Uint8Array([
  0x59, 0x4a, 0x3b, 0x2c, 0x1d, 0x0e, 0xff, 0xf0,
  0xe1, 0xd2, 0xc3, 0xb4, 0xa5, 0x96, 0x87, 0x78,
  0x69, 0x5a, 0x4b, 0x3c, 0x2d, 0x1e, 0x0f, 0x00,
  0xf1, 0xe2, 0xd3, 0xc4, 0xb5, 0xa6, 0x97, 0x88,
]);

/**
 * Generates a random 32-byte blinding factor.
 */
function randomBlinding(): Uint8Array {
  return crypto.getRandomValues(new Uint8Array(32));
}

async function proveAndVerifyTransferAmount(transferAmount: bigint): Promise<void> {
  const r = randomBlinding();

  // Generate the proof.
  const { proof, comms } = await batchRangeProof({
    v: [transferAmount],
    rs: [r],
    valBase: VAL_BASE,
    randBase: RAND_BASE,
    numBits: 64, // prove transferAmount is in [0, 2^64)
  });

  console.log(`Proof length: ${proof.byteLength} bytes`);
  console.log(`Commitment (hex): ${Buffer.from(comms[0]).toString('hex')}`);

  // Verify the proof locally before submitting on-chain.
  const valid = await batchVerifyProof({
    proof,
    comms,
    valBase: VAL_BASE,
    randBase: RAND_BASE,
    numBits: 64,
  });

  if (!valid) {
    throw new Error('Range proof failed local verification — this should never happen');
  }

  console.log('Proof verified. Ready to submit on-chain.');
  // Submit proof and comms[0] as part of your Aptos transaction payload.
}

await proveAndVerifyTransferAmount(1_000_000n);
```

## 3. Batch proving multiple values

A single Bulletproof can cover multiple values simultaneously. This is more efficient than generating one proof per value and is required when a transaction involves several confidential amounts (for example, splitting a balance across multiple recipients).

```typescript
import { batchRangeProof, batchVerifyProof } from '@aptos-labs/confidential-asset-bindings';

const VAL_BASE = new Uint8Array(32).fill(1);  // replace with real base point
const RAND_BASE = new Uint8Array(32).fill(2); // replace with real base point

function randomBlinding(): Uint8Array {
  return crypto.getRandomValues(new Uint8Array(32));
}

async function batchProveAmounts(amounts: bigint[]): Promise<{
  proof: Uint8Array;
  comms: Uint8Array[];
}> {
  if (amounts.length > 16) {
    throw new Error('Maximum batch size is 16');
  }

  const blindings = amounts.map(() => randomBlinding());

  const { proof, comms } = await batchRangeProof({
    v: amounts,
    rs: blindings,
    valBase: VAL_BASE,
    randBase: RAND_BASE,
    numBits: 64,
  });

  // Verify before returning.
  const valid = await batchVerifyProof({
    proof,
    comms,
    valBase: VAL_BASE,
    randBase: RAND_BASE,
    numBits: 64,
  });

  if (!valid) {
    throw new Error('Batch range proof failed verification');
  }

  console.log(`Proved ${amounts.length} values in a single proof (${proof.byteLength} bytes)`);
  return { proof, comms };
}

// Prove three amounts at once.
const { proof, comms } = await batchProveAmounts([500n, 1500n, 75000n]);
// comms[0] commits to 500, comms[1] to 1500, comms[2] to 75000.
```

The maximum batch size is 16 values. `comms[i]` corresponds to `v[i]`, in the same order.

## 4. Integrating in a React Native / Expo app

No additional setup is required for React Native or Expo. The package automatically uses the native JSI module when running on a device or in the Expo Go / custom dev client. The import is identical to Web and Node.js.

```typescript
import React, { useState } from 'react';
import { Button, Text, View } from 'react-native';
import {
  solveDiscreteLog,
  batchRangeProof,
  batchVerifyProof,
} from '@aptos-labs/confidential-asset-bindings';

const VAL_BASE = new Uint8Array(32).fill(1);  // replace with real base point
const RAND_BASE = new Uint8Array(32).fill(2); // replace with real base point

export default function ConfidentialAssetDemo() {
  const [status, setStatus] = useState<string>('Idle');

  async function handleProve() {
    setStatus('Generating proof...');
    try {
      const r = crypto.getRandomValues(new Uint8Array(32));
      const { proof, comms } = await batchRangeProof({
        v: [42n],
        rs: [r],
        valBase: VAL_BASE,
        randBase: RAND_BASE,
        numBits: 32,
      });

      const valid = await batchVerifyProof({
        proof,
        comms,
        valBase: VAL_BASE,
        randBase: RAND_BASE,
        numBits: 32,
      });

      setStatus(valid ? `Proof valid (${proof.byteLength} bytes)` : 'Proof invalid');
    } catch (err) {
      setStatus(`Error: ${(err as Error).message}`);
    }
  }

  return (
    <View style={{ flex: 1, justifyContent: 'center', alignItems: 'center' }}>
      <Text>{status}</Text>
      <Button title="Generate & Verify Proof" onPress={handleProve} />
    </View>
  );
}
```

The native module is initialized automatically. There is no explicit `init()` or `load()` call needed on React Native.

## 5. Node.js script

A minimal Node.js script that exercises both `solveDiscreteLog` and `batchRangeProof`. The WASM binary is loaded from `node_modules`; if that fails, the library falls back to the unpkg CDN.

```typescript
// scripts/demo.ts  (run with: npx tsx scripts/demo.ts)
import {
  solveDiscreteLog,
  batchRangeProof,
  batchVerifyProof,
} from '@aptos-labs/confidential-asset-bindings';

const VAL_BASE = new Uint8Array(32).fill(1);  // replace with real base point
const RAND_BASE = new Uint8Array(32).fill(2); // replace with real base point

async function main() {
  // --- Range proof ---
  console.log('Generating range proof...');
  const r = crypto.getRandomValues(new Uint8Array(32));

  const { proof, comms } = await batchRangeProof({
    v: [999n],
    rs: [r],
    valBase: VAL_BASE,
    randBase: RAND_BASE,
    numBits: 32,
  });
  console.log(`  Proof: ${proof.byteLength} bytes`);
  console.log(`  Commitment: ${Buffer.from(comms[0]).toString('hex')}`);

  const valid = await batchVerifyProof({
    proof,
    comms,
    valBase: VAL_BASE,
    randBase: RAND_BASE,
    numBits: 32,
  });
  console.log(`  Verification: ${valid}`);

  // --- Discrete log (small value for demo speed) ---
  // In practice, encryptedPoint comes from an on-chain account resource.
  // Here we show the call signature; the point below is illustrative only.
  const encryptedPoint = new Uint8Array(32);
  encryptedPoint[0] = 0x01; // placeholder — use a real on-chain point

  try {
    const balance = await solveDiscreteLog(encryptedPoint, 16);
    console.log(`Decrypted balance: ${balance}`);
  } catch (err) {
    console.error(`solveDiscreteLog: ${(err as Error).message}`);
  }
}

main().catch(console.error);
```

## 6. Common patterns

### Generating a random blinding factor

A blinding factor is a 32-byte uniformly random scalar. Use the Web Crypto API, which is available in all supported environments (browsers, Node.js ≥ 19, React Native via polyfill or native `crypto`):

```typescript
function randomBlinding(): Uint8Array {
  return crypto.getRandomValues(new Uint8Array(32));
}
```

**Important:** each proof generation call requires a fresh blinding factor per value. Reusing a blinding factor across different values with the same base points leaks information about the secret values.

### Using `comms` in an on-chain transaction

The `comms` array returned by `batchRangeProof` contains the Pedersen commitments that must be submitted on-chain alongside the proof. Each commitment is a 32-byte compressed Ristretto255 point. Pass them as raw bytes in your transaction payload:

```typescript
import {
  batchRangeProof,
  type BatchRangeProofResult,
} from '@aptos-labs/confidential-asset-bindings';

const VAL_BASE = new Uint8Array(32).fill(1);  // replace with real base point
const RAND_BASE = new Uint8Array(32).fill(2); // replace with real base point

async function buildTransferPayload(transferAmount: bigint) {
  const r = crypto.getRandomValues(new Uint8Array(32));

  const { proof, comms }: BatchRangeProofResult = await batchRangeProof({
    v: [transferAmount],
    rs: [r],
    valBase: VAL_BASE,
    randBase: RAND_BASE,
    numBits: 64,
  });

  // comms[0] is the Pedersen commitment to transferAmount.
  // Both proof and comms[0] are submitted as part of the on-chain transaction.
  return {
    proofBytes: proof,        // Uint8Array — serialized Bulletproof
    commitment: comms[0],     // Uint8Array (32 bytes) — public commitment
  };
}
```

The commitment at `comms[i]` binds to `v[i]` with blinding `rs[i]`. The on-chain verifier checks that the supplied proof is consistent with the supplied commitments. Keep the blinding factor `r` secret; it is not submitted on-chain.

### Handling errors defensively

```typescript
import { solveDiscreteLog } from '@aptos-labs/confidential-asset-bindings';

async function safeDecrypt(point: Uint8Array): Promise<bigint | null> {
  if (point.byteLength !== 32) {
    console.error('Invalid point: must be 32 bytes');
    return null;
  }

  try {
    return await solveDiscreteLog(point, 32);
  } catch (err) {
    const message = (err as Error).message;
    if (message.includes('confidential asset operation failed')) {
      // Internal error — retry or surface to user
      console.error('Internal cryptographic failure');
    } else {
      // Validation error — message contains specifics
      console.error(`Validation error: ${message}`);
    }
    return null;
  }
}
```
